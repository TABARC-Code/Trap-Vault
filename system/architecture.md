# System Architecture

This is not a model.

It’s a structure around a model.

---

## Flow

Input → Classification → Context → Reasoning → Output

---

## Key Idea

The model does not think globally.

It thinks inside context.

---

## Components

### Classification
Determines where something belongs.

### Taxonomy
Defines structure and relationships.

### Reasoning
Handled by Claude, but constrained.

### Routing
Handles ambiguity and weighting.

---

## Failure Mode

Wrong classification → wrong context → wrong answer

The system fails at placement, not reasoning.
